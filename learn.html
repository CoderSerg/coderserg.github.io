Introduction to Cybersecurity

Welcome! This tutorial teaches you safe, legal cybersecurity. You’ll learn how attackers think, how to stay protected, and how to test your own systems.

How Hackers Think

Hackers think differently from normal users. They look for:

  • Weak input fields
  • Bad passwords
  • Unprotected admin pages
  • Misconfigured servers

Learning the mindset helps you defend your own systems better.

Safe Testing Environment

Ethical hackers always test inside a safe environment, such as:

  • Virtual machines (VirtualBox, VMware)
  • Kali Linux or Parrot OS
  • DVWA (Damn Vulnerable Web App)
  • Localhost-only testing

Never test on real websites or networks that you don’t own.

Simple Attacks Explained

Here are basic examples hackers use — but ONLY in safe environments:

  • SQL Injection (like `' OR 1=1 --`)
  • XSS (Cross-Site Scripting)
  • Password brute forcing
  • Packet sniffing on your own network

More advanced examples will come in future lessons.
